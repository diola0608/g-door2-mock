# 03. ページ追加プレイブック（手順とチェックリスト）

> 新しいページ／導線を、このmockの設計思想を保ったまま追加するための手順書。
> 前提：[00-principles.md](00-principles.md)（思想）・[01-domain-model.md](01-domain-model.md)（背景）・
> [02-component-catalog.md](02-component-catalog.md)（部品）を読んでいること。

---

## 手順

### Step 0 ── 文脈を読み込む
`docs/00–03` に目を通す。既存ページのうち**作るものに最も近い1枚**を開いて構造を掴む
（一覧系→`roster.html`／詳細系→`roster-detail.html`／設定系→`attributes.html`・`codes.html`／
ウィザード→`import.html`／承認キュー→`transfers.html`／作成系→`form-builder.html`／
ダッシュボード→`index.html`／権限→`permissions.html`）。

### Step 1 ── 深掘りする（最重要）
[00 の「深掘り5問」](00-principles.md#1-着手前の深掘り5問) に**自分で答える**。
誰が・どんな業務で・動線全体・常時可視化すべき状態・失敗系。答えがそのまま構成要素になる。
ユーザーが言っていなくても、ドメイン上「在るべき」画面・状態・導線は自分から補完する。

### Step 2 ── （複数ページなら）動線を先に提案する
作るのが複数ページや新サービスなら、**いきなり実装せず**、ページ一覧と遷移（どこから入り、
どこへ抜けるか）を1〜2文ずつで先に提示し、合意を取ってから作る。1ページでも入口/出口は決める。

### Step 3 ── 雛形から作る
- `_page-template.html` を複製して新ファイル名にする（例 `audit-log.html`）。
- `<title>` と breadcrumb と page-head を埋める。
- `<body data-page="X">` の `X` を決める。
- `assets/app.js` の `NAV` 配列に項目を追加（id は `X` と一致）。必要なら新セクションを作る。
  → [02 §3](02-component-catalog.md#3-レイアウトシェル自動注入と-nav-への項目追加) の手順どおり。

### Step 4 ── カタログの部品だけで本体を組む
- 見た目は [02](02-component-catalog.md) のクラスで構成。**独自CSSはレイアウト用gridと小装飾に限定**。
- 該当するドメイン複合パターン（権限文章トークン／マスクセル／自分の権限ペイン／組織ツリー／
  カテゴリタブ／CSVウィザード／属性履歴／「いつ反映」表現…）を [02 §6](02-component-catalog.md#6-ドメイン複合パターンレシピ新ページでも一貫させる) から再利用。
- ダミーデータは [01 §14](01-domain-model.md#14-既定ペルソナダミーの視点) のペルソナ・地理・ID規則でリアルに。

### Step 5 ── UIインタラクションを配線
- ボタン→モーダルは `Composer.openModal/closeModal`、操作結果は `Composer.toast`。
- タブは `data-tabs`/`data-tab-panel` を付けるだけ（JS不要）。
- ページ固有のミニ挙動が要るなら末尾に小さな `<script>`（例：ウィザードの `wiz.go`）。

### Step 6 ── セルフレビュー（[00 §5](00-principles.md#5-出力前のセルフチェック合格ライン) を実施）
権限スコープ／同期／監査／非同期反映／エラー・マスク／breadcrumb・導線／色・アイコンの意味整合／
リアルなダミーデータ／既存部品で構成／レスポンシブ。全部 YES になるまで直す。

## 「LLMに何を渡すか」（運用の肝）

> ❌ 毎回 mock を丸ごと渡す → コストが膨張・再現がぶれる・*なぜ*が伝わらない。
> ✅ 下記の最小セットを渡す。

| 使うツール | 渡すもの |
|---|---|
| **エージェント系**（Claude Code, Cursor 等／リポにアクセスできる） | 何も貼らなくてよい。リポ常駐の [`AGENTS.md`](../AGENTS.md) が `docs/` と `assets/` を参照させる。依頼文だけ書く。 |
| **チャット系**（リポにアクセスできない） | [`docs/04-chat-llm-prompt.md`](04-chat-llm-prompt.md) を貼り、続けて `00`–`03` と **最も近い既存ページ1枚** の中身を添付。**全mockは不要。** |

新しい *種類* のページ（既存に近いものが無い）を作るときだけ、近い既存ページを1〜2枚追加で見せる。

## ありがちな失敗と回避

- サイドバーに出てこない → `app.js` の `NAV` に追加したか、`data-page` の id が一致しているか確認。
- ヘッダー/サイドバーが二重・出ない → `<main>` を1つにし、`app.js` を**末尾**で読み込む。骨格HTMLを手書きしない。
- 見た目が他ページと違う → 独自CSSで作っていないか。まず [02](02-component-catalog.md) のクラスを探す。
- 権限/同期/履歴の気配が無い → [00 §3](00-principles.md#3-このmockを良くしている署名的挙動必ず踏襲する6つ) の6挙動を入れ忘れている。
