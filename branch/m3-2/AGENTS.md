## 要件プロンプト（`prompt/`）に基づく開発

`prompt/` には、モック構築の元となった要件が番号付きで格納されています（例：`001-初期構築.txt`）。
**どの番号まで適用済みかは [`prompt/applied-history.md`](prompt/applied-history.md) が唯一の正**である。
ユーザーに口頭で確認しない。

### プロンプトと成果物の関係

| 種類 | 役割 |
|---|---|
| **`prompt/NNN-*.txt`** | 番号付き要件本文。 |
| **`prompt/applied-history.md`** | 適用済み番号の履歴（日時・LLM・番号）。**着手前に必ず読む。** |
| **`docs/00`–`03`** | 適用済みプロンプトをもとに蒸留した運用ドキュメント。日常の拡張ではこちらが**実装の正**。 |

### 「プロンプト N を実行してください」と指示されたとき

**実装に入る前に、必ず `applied-history.md` を確認し、次のゲートを通過すること。**

1. **N が既に適用済みか** … 履歴に N がある → **「プロンプト N は実装済みです」** と伝えて**処理を停止**する（再実装しない）。
2. **N より小さい番号がすべて適用済みか** … `001` 〜 `N-1` のいずれかが履歴にない →
   **未適用の番号を列挙して伝え、処理を停止**する（飛び番号で適用しない）。
3. 上記を通過したら、`prompt/` 内の **N 番**のファイルを読み、それを**今回の作業要件**とする。
4. **N より大きい番号** … ユーザーが編集中の未適用要件。**読まない・参照しない・実装に反映しない**。
5. 作業は [`docs/03`](docs/03-add-a-page-playbook.md) の手順と、下記「絶対に守ること」に従う。
6. **適用完了後** … `applied-history.md` に 1 行追加する：
   - **いつ** … 適用完了日時（`YYYY-MM-DD HH:MM`、ローカル時刻でよい）
   - **どの LLM が** … 自分のモデル名（例：`Composer`、`Claude Sonnet 4.6`）。不明なら `（不明）`
   - **何番を** … プロンプト番号（3 桁ゼロ埋め、例：`002`）

### 番号指定がないとき

1. `applied-history.md` から適用済みの最大番号を把握する。
2. 適用済みプロンプトと `docs/`・既存モックを正とする。
3. **適用済み最大番号より大きい** `prompt/` のファイルは読まない。

## 絶対に守ること
- **思想を再現する**：要件はざっくり来る。質問で埋めず、ユーザーの背景・現場・権限環境を
  自分で深く推測し、必要な画面・状態・導線を一式設計して作り切る。「動く」でなく「分かる」を作る。
  docs/00 の署名的挙動①〜⑥（権限文章＋シミュレーター／マスク・自分の権限／非同期バッチ／履歴／
  一方向同期／リアルなダミーデータ）を該当箇所で必ず入れる。
- **資産を再利用する（ソース・オブ・トゥルース）**：
  - `assets/base.css` … デザイントークン＋全コンポーネント。**新しい見た目は、まずここにあると考える。**
  - `assets/app.js` … ヘッダー/サイドバーの自動注入＋`Composer.openModal/closeModal/toast`＋タブ等の配線。
  - コンポーネントを**再発明しない**。独自CSSはレイアウト用gridと小装飾に限定。
- **骨格を壊さない**：各ページは `<main>` の中身だけ書く。`<body data-page="X">` を設定し、
  `assets/app.js` を**末尾**で読み込む。ヘッダー/サイドバーのHTMLは手書きしない。
- **サイドバーに載せる**：新ページは `assets/app.js` の `NAV` 配列に項目を1つ追加（id を `data-page` と一致）。
  これは唯一許容される共有ファイルの編集。

## 進め方（docs/03 準拠）
深掘り5問に自答 → （複数ページなら）動線を先に提案 → `_page-template.html` を複製して作成 →
`NAV` 追加 → docs/02 の部品で構築 → `Composer.*` で配線 → docs/00 のセルフチェックで仕上げ。

## 完了時
- どんなユーザー・動線を想定し、権限/同期/履歴をどう見せたかを数行で要約。
- 追加した `NAV` 項目（id/icon/label/href）を明記。
- 可能なら新ページをブラウザで開き、サイドバー項目・タブ・モーダル・トーストの動作を確認。

> 既存の良い実例：`audit-log.html`（このキットの作法だけで作られた監査ログ画面）。迷ったら参照。
