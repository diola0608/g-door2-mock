rule取込・連携設定

CSV/TSV やクリップボード貼り付けで受領するデータを、現場のファイルのまま取り込めるようにする設定です。ヘッダーの自動判定・本人同定キー・変換ルールを事前に整備します。設定はCSVインポートウィザードで使われます。
upload_file取込を開始
match_word
取込時にファイルのヘッダー行を判定文字列と突き合わせ、名簿項目へ自動マッピングします。1つの項目に判定文字列を複数登録でき、サードパーティごとに異なる列名（「Email / メール / mailaddress」など）を吸収します。判定できなかった列はウィザードのステップ2で手動指定します。

fingerprint
取込データが誰のデータかを判別するためのキーです。初めて一致したときに登録し、次回以降は同じキーで既存の人物データへ紐付けます。列名のゆらぎは判定文字列で吸収します。

badge
人と1対1 でないデータに注意: データは「人と1対1」のものと、「人の所属と1対1」のものがあります。例えば X さんは A小学校では校長、B小学校では教頭 という具合に、所属先によって役職が異なります。この場合は識別キー（人）＋ 組織名／組織コードで所属レコードを特定して紐付けます。
transform
受領したファイルを、名簿の項目へそのままマッピングできない場合に変換します。ファイル全体の前処理（メタ行除去など）と、1レコードごとの変換（姓名分割・値の読み替え・コード変換）を、正規表現や JavaScript で柔軟に定義できます。
descriptionファイル全体の前処理 取込前にファイル全体へ適用

例: サードパーティ出力のCSVで、先頭3行にシステム名・更新日などのファイル情報が入っているケース。これらを除去してからヘッダー判定にかけます。

// 先頭3行のメタ情報（システム名・更新日時）を除去し、ヘッダー行を先頭にする function preprocess(rows) { const meta = rows.slice(0, 3); // 例: ["校務システムX", "更新日,2026-06-12", ...] log("source", meta[0][0]); // 監査ログへ出力元を記録 return rows.slice(3) // 4行目以降を採用 .map(r => r.map(c => c.replace(/　/g, " ").trim())); // 全角空白→半角・trim }

code
変換は正規表現またはDBに登録した JavaScriptで実行されます。組織タイプ・所属自治体などの条件で分岐でき、適用結果は実行ジョブのログに記録されます（個人情報はマスク）。
output
各サードパーティへ連携するため、インポートとは逆向きの変換を行います。アプリ内の値を標準コードへ変換し、分割している項目を結合するなどしてから書き出します。連携は Composer → 外部システム の一方向です。
